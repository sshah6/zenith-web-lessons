3- How can a developer use a Google font?
You can choose one of 2 methods to use a google/web font. Option 1 Online font services generally store and serve fonts for you so you don't have to
worry about writing the @font-face code. Instead, you generally just need to
insert a simple line or two of code into your site to make everything work.
Examples include Adobe Fonts and Cloud.typography. Most of these services are
subscription-based, with the notable exception of Google Fonts, a useful
free service, especially for rapid testing work and writing demos.
On the html page in the header link to the font with a link elelment above the css
stylesheet. (So it will load into the CSS stylesheet)

//< link href="https://fonts.googleapis.com/css2?family=Comforter&family=Open+Sans:wght@300&family=Quintessential&display=swap" rel="stylesheet">

After this you can call the desired font in the CSS style sheet.

//h1{ //font: Tangerine; //}

Option 2 To store the font locally you can download a .zip file of your desired font. Store
this zip file in the same directory as your css/html files. On your stylesheet.
css at the very top of the page you will need to insert "@font-face rulesets"

//@font-face { //font-family: "zantrokeregular"; //src: url("fonts/zantroke-webfont.woff2") format("woff2"), url("fonts/
zantroke-webfont.woff") //format("woff"); //font-weight: normal; // font-style: normal; // } //font-family: "zantrokeregular", serif;

4- How can a developer figure out what font is being used on a given web page?

Answer4. Open the console. Use the Elements tab. Select the text (body) in the
console. Click on the Computed tab in the lower portion of the console and
search for "font-family." To see the CSS related to the font, look under the
Styles tab. You can scroll through to search for font-related values.
However, as there might be overrides and irrelevant style rules here, the
Computed tab is usually more helpful.
